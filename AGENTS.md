# Git Workflow Policy

- **Never push directly to `main`.** The `main` branch is protected and all changes must go through pull requests.

- **Always create a feature branch** for any work. Branch off `main` and use a descriptive name (e.g., `fix/login-error`, `feat/user-dashboard`).

- **Open a pull request on GitHub** before merging. All PRs require review before they can be merged into `main`.
